Tenant anlegen

Zuerst die App-Registrierung im Microsoft-Tenant vorbereiten, dann die Werte hier eintragen und Admin Consent erteilen.

1. App-Registrierung im Entra Admin Center

  1. Öffne entra.microsoft.com (oder Azure Portal → App registrations) und melde dich als Global Administrator oder Application Administrator im Kunden-Tenant an.
  2. Gehe zu Identity → Applications → App registrations → New registration.
  3. Name z.B. M365 Backup. Unter Supported account types: Accounts in this organizational directory only (Single tenant). Redirect URI vorerst leer lassen → Register.
  4. Auf der Übersicht notieren:
    • Application (client) ID → Feld unten
    • Directory (tenant) ID → Feld Azure Tenant ID

2. Client Secret erzeugen

  1. Certificates & secrets → Client secrets → New client secret.
  2. Beschreibung z.B. m365backup, Gültigkeit möglichst 12–24 Monate.
  3. Value sofort kopieren (nur einmal sichtbar) → Feld Client secret. Ablaufdatum → Feld Secret expires.

Token-Renewal: Access-Tokens (OAuth) erneuert die App automatisch über Client Credentials. Das Client Secret selbst wird aktuell nicht automatisch rotiert — die App warnt bei Ablauf (30/7 Tage). Vor Ablauf manuell ein neues Secret anlegen und hier aktualisieren.

Nach dem Hinzufügen: Grant admin consent for <Tenant> im Portal klicken oder nach dem Speichern hier den Button „Admin consent“ am Tenant nutzen.

4. Redirect URI für Admin Consent

  1. Authentication → Add a platform → Web.
  2. Redirect URI eintragen (exakt, inkl. https): {{.RedirectURI}}
  3. Speichern. Keine weiteren Redirects nötig.

5. Werte hier eintragen

  1. Nach dem Anlegen: Tenant öffnen → Admin consent → Kunde bestätigt die Permissions.
  2. Status wird active, erste Backup-Jobs starten automatisch.

Secrets werden mit MASTER_KEY verschlüsselt in der Datenbank gespeichert — nie im Klartext ins Git.
